# Requisitos e Telas

## Proposta

Assim como solicitado, o objetivo inicial do trabalho é a implementação das telas do protótipo interativo, o qual abrange os requisitos mantidos do
documento

## Requisitos

- Autenticar Vínculo Institucional (UEM);
- Cadastrar Usuário Externo;
- Autenticar Usuário Externo;
- Autenticar Administradores;
- Validar CPF de Usuários Externos;
- Identificar Tipo de Usuário;
- Apresentar Cardápio Diário;
- Exibir Avisos Recentes;
- Consultar Avisos;
- Apresentar Cardápio Semanal;
- Enviar Notificações;
- Verificar Saldo de Fichas;
- Manter Histórico de Pagamento;
- Manter Histórico de Consumo;
- Comprar Fichas;
- Selecionar e Quantificar Fichas;
- Validar Quantidade de Fichas;
- Calcular Preço Dinâmico;
- Selecionar Opção de Pagamento;
- Processar Transação;
- Pagar com Cartão;
- Pagar com PIX;
- Atualizar Status Pós-Pagamento;
- Gerar QR Codes;
- Exibir QR Code Condicionalmente;
- Consumir Ficha;
- Coletar Avaliações;
- Gerenciar Status das Avaliações;
- Registrar Avaliações;
- Cadastrar e Publicar Cardápio Semanal;
- Visualizar Cardápio Ativo;
- Alterar Item do Cardápio Ativo;
- Criar e Publicar Aviso;
- Visualizar e Gerenciar Avisos;
- Alterar e Remover Avisos;
- Gerar Relatório de Avaliações;
- Gerar Relatório de Frequência;
- Gerar Relatório de Recebimento de Fichas;
- Filtrar Dados de Relatórios;
- Integrar Dados Institucionais;
- Persistir Dados.
